Verify Initial Snapshot in Snowflake

Confirm that the initial data has been replicated from Postgres to Snowflake.

Switch to a Snowflake worksheet to verify the initial data replication.

⚠
Case Sensitivity

PostgreSQL uses lowercase identifiers by default. When querying replicated data in Snowflake, you MUST quote lowercase schema and table names: "ecommerce"."customers"

Check Schema and Tables Were Created

SQL (Snowflake) 
-- Check that the ecommerce schema was created
SHOW SCHEMAS IN DATABASE ECOMMERCE_CDC;

-- Check that tables were created
SHOW TABLES IN SCHEMA ECOMMERCE_CDC."ecommerce";
✅ Expected Result

You should see the ecommerce schema and three tables: customers, products, orders.

Verify Row Counts

SQL (Snowflake) 
-- Verify row counts match the source
SELECT 'customers' AS table_name, COUNT(*) AS row_count
FROM ECOMMERCE_CDC."ecommerce"."customers"
UNION ALL
SELECT 'products', COUNT(*)
FROM ECOMMERCE_CDC."ecommerce"."products"
UNION ALL
SELECT 'orders', COUNT(*)
FROM ECOMMERCE_CDC."ecommerce"."orders";
✅ Expected Result
TABLE_NAME  | ROW_COUNT
------------+-----------
customers   |        10
products    |        10
orders      |        10

Preview Replicated Data

SQL (Snowflake) 
-- Preview customers data
SELECT * FROM ECOMMERCE_CDC."ecommerce"."customers" LIMIT 5;

-- Preview products data
SELECT * FROM ECOMMERCE_CDC."ecommerce"."products" LIMIT 5;

-- Preview orders data
SELECT * FROM ECOMMERCE_CDC."ecommerce"."orders" LIMIT 5;
✅ Expected Result

All tables contain the same data you inserted in Step 3. The initial snapshot replication is complete.
