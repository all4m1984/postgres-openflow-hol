Test Real-Time CDC (INSERT, UPDATE, DELETE)

Make changes in Postgres and verify they're captured in real-time in Snowflake.

Switch back to your psql terminal connected to Snowflake Postgres.

Test INSERT: Add New Customers and Orders

SQL (PostgreSQL) 
-- Insert new customers
INSERT INTO ecommerce.customers (first_name, last_name, email, city) VALUES
('Kevin', 'Hart', 'kevin@comedy.com', 'Philadelphia'),
('Laura', 'Chen', 'laura@tech.com', 'San Francisco'),
('Marcus', 'Williams', 'marcus@sports.com', 'Miami');

-- Insert new orders
INSERT INTO ecommerce.orders (order_date, customer_id, product_id, quantity, status) VALUES
('2025-01-25', 11, 2, 1, 'pending'),
('2025-01-25', 12, 5, 2, 'pending'),
('2025-01-26', 13, 3, 1, 'processing');

Test UPDATE: Modify Existing Records

SQL (PostgreSQL) 
-- Update order statuses (simulating order fulfillment)
UPDATE ecommerce.orders SET status = 'shipped' WHERE order_id = 7;
UPDATE ecommerce.orders SET status = 'completed' WHERE order_id = 10;

-- Update product stock after orders
UPDATE ecommerce.products SET stock_quantity = stock_quantity - 1 WHERE product_id = 2;
UPDATE ecommerce.products SET stock_quantity = stock_quantity - 2 WHERE product_id = 5;

-- Update customer email
UPDATE ecommerce.customers SET email = 'alice.smith@newemail.com' WHERE customer_id = 1;

Test DELETE: Remove a Record

SQL (PostgreSQL) 
-- Cancel an order (delete)
DELETE FROM ecommerce.orders WHERE order_id = 11;
ⓘ
CDC Latency

Changes are typically replicated within seconds to 1 minute depending on your merge task schedule. Wait approximately 60 seconds before verifying in Snowflake.

ⓘ
How the connector handles DELETE operations

The Openflow PostgreSQL connector uses soft deletes — deleted rows are never physically removed from Snowflake. Instead, the connector sets the metadata column _SNOWFLAKE_DELETED = TRUE on the row. Every replicated row also includes _SNOWFLAKE_INSERTED_AT and _SNOWFLAKE_UPDATED_AT metadata columns. This is expected and correct behavior that preserves full change history.

Verify Changes in Snowflake

Switch to your Snowflake worksheet and verify:

SQL (Snowflake) 
-- Verify INSERT: New customers (exclude soft-deleted rows)
SELECT COUNT(*) AS total_active_customers
FROM ECOMMERCE_CDC."ecommerce"."customers"
WHERE "_SNOWFLAKE_DELETED" = FALSE;

-- Verify the new customers
SELECT "customer_id", "first_name", "last_name", "city"
FROM ECOMMERCE_CDC."ecommerce"."customers"
WHERE "customer_id" > 10;

-- Verify UPDATE: Check order status changes
SELECT "order_id", "status" FROM ECOMMERCE_CDC."ecommerce"."orders"
WHERE "order_id" IN (7, 10);

-- Verify UPDATE: Check Alice's updated email
SELECT "first_name", "last_name", "email" FROM ECOMMERCE_CDC."ecommerce"."customers"
WHERE "customer_id" = 1;

-- Verify UPDATE: Check product stock quantities
SELECT "product_name", "stock_quantity" FROM ECOMMERCE_CDC."ecommerce"."products"
WHERE "product_id" IN (2, 5);

-- Verify DELETE: Order 11 is soft-deleted (_SNOWFLAKE_DELETED = TRUE)
SELECT "order_id", "_SNOWFLAKE_DELETED", "_SNOWFLAKE_UPDATED_AT"
FROM ECOMMERCE_CDC."ecommerce"."orders"
WHERE "order_id" = 11;

-- Count only active (non-deleted) orders
SELECT COUNT(*) AS active_orders
FROM ECOMMERCE_CDC."ecommerce"."orders"
WHERE "_SNOWFLAKE_DELETED" = FALSE;
