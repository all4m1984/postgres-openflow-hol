Enable Logical Replication for CDC

Configure PostgreSQL WAL and create a publication for Change Data Capture.

PostgreSQL CDC requires logical replication to be enabled. We need to verify the WAL level and create a publication for the tables we want to replicate.

ⓘ
Snowflake Postgres Default

Snowflake Postgres instances come with wal_level = logical already enabled. You can verify this but do not need to change it.

Verify WAL Level

SQL (PostgreSQL) 
-- Verify logical replication is enabled
SHOW wal_level;
✅ Expected Result
 wal_level
-----------
 logical

Create a Publication for CDC

A publication defines which tables should be tracked for changes:

SQL (PostgreSQL) 
-- Create a publication for all ecommerce tables
CREATE PUBLICATION ecommerce_cdc_publication
  FOR TABLE
    ecommerce.customers,
    ecommerce.products,
    ecommerce.orders;
✅ Expected Result
CREATE PUBLICATION

Verify the Publication

SQL (PostgreSQL) 
-- List all publications
SELECT * FROM pg_publication;

-- List tables in our publication
SELECT * FROM pg_publication_tables
WHERE pubname = 'ecommerce_cdc_publication';
✅ Expected Result
         pubname              | pubowner | puballtables | pubinsert | pubupdate | pubdelete
------------------------------+----------+--------------+-----------+-----------+-----------
 ecommerce_cdc_publication   |       10 | f            | t         | t         | t

       pubname               | schemaname | tablename
-----------------------------+------------+-----------
 ecommerce_cdc_publication   | ecommerce  | customers
 ecommerce_cdc_publication   | ecommerce  | products
 ecommerce_cdc_publication   | ecommerce  | orders

Verify Replica Identity

For tables with primary keys, the connector requires REPLICA IDENTITY DEFAULT (the PostgreSQL default). Verify your tables are correctly set:

SQL (PostgreSQL) 
-- Verify replica identity is DEFAULT for all tables (required by Openflow connector)
SELECT relname AS table_name,
       CASE relreplident
           WHEN 'd' THEN 'DEFAULT'
           WHEN 'n' THEN 'NOTHING'
           WHEN 'f' THEN 'FULL'
           WHEN 'i' THEN 'USING INDEX'
       END AS replica_identity
FROM pg_class
WHERE relname IN ('customers', 'products', 'orders');
⚠
Do NOT set REPLICA IDENTITY FULL for tables with primary keys

The Openflow PostgreSQL connector requires REPLICA IDENTITY DEFAULT for tables with primary keys. Setting it to FULL will cause the connector to fail with MISSING_PRIMARY_KEYS and permanently stop replication for those tables. DEFAULT is already the PostgreSQL default — no change is needed if you have not altered it.
