Run Analytics on Replicated Data

Demonstrate the value of real-time data with analytical queries in Snowflake.

Query 1: Real-Time Order Dashboard

SQL (Snowflake) 
-- Real-time order status dashboard
SELECT
    o."status" AS order_status,
    COUNT(*) AS order_count,
    SUM(p."price" * o."quantity") AS total_revenue
FROM ECOMMERCE_CDC."ecommerce"."orders" o
JOIN ECOMMERCE_CDC."ecommerce"."products" p
    ON o."product_id" = p."product_id"
GROUP BY o."status"
ORDER BY total_revenue DESC;

Query 2: Customer Lifetime Value (CLV)

SQL (Snowflake) 
-- Customer lifetime value analysis
SELECT
    c."first_name" || ' ' || c."last_name" AS customer_name,
    c."city",
    COUNT(o."order_id") AS total_orders,
    SUM(p."price" * o."quantity") AS lifetime_value,
    MIN(o."order_date") AS first_order,
    MAX(o."order_date") AS last_order
FROM ECOMMERCE_CDC."ecommerce"."customers" c
LEFT JOIN ECOMMERCE_CDC."ecommerce"."orders" o
    ON c."customer_id" = o."customer_id"
LEFT JOIN ECOMMERCE_CDC."ecommerce"."products" p
    ON o."product_id" = p."product_id"
GROUP BY c."first_name", c."last_name", c."city"
ORDER BY lifetime_value DESC NULLS LAST;

Query 3: Product Performance with Inventory Alerts

SQL (Snowflake) 
-- Product performance with low stock alerts
SELECT
    p."product_name",
    p."category",
    p."price",
    p."stock_quantity",
    COALESCE(SUM(o."quantity"), 0) AS units_sold,
    COALESCE(SUM(p."price" * o."quantity"), 0) AS revenue,
    CASE
        WHEN p."stock_quantity" < 50 THEN 'LOW STOCK'
        WHEN p."stock_quantity" < 100 THEN 'MEDIUM'
        ELSE 'HEALTHY'
    END AS inventory_status
FROM ECOMMERCE_CDC."ecommerce"."products" p
LEFT JOIN ECOMMERCE_CDC."ecommerce"."orders" o
    ON p."product_id" = o."product_id"
GROUP BY p."product_name", p."category", p."price", p."stock_quantity"
ORDER BY revenue DESC;

Query 4: Daily Revenue Trend

SQL (Snowflake) 
-- Daily revenue trend with cumulative total
SELECT
    o."order_date",
    COUNT(*) AS orders_placed,
    SUM(p."price" * o."quantity") AS daily_revenue,
    SUM(SUM(p."price" * o."quantity")) OVER (ORDER BY o."order_date") AS cumulative_revenue
FROM ECOMMERCE_CDC."ecommerce"."orders" o
JOIN ECOMMERCE_CDC."ecommerce"."products" p
    ON o."product_id" = p."product_id"
GROUP BY o."order_date"
ORDER BY o."order_date";
✅ Expected Result

A day-by-day revenue trend showing order activity and cumulative revenue growth, demonstrating how real-time replicated data enables immediate analytics without impacting the source OLTP system.
