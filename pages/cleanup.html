Cleanup (Optional)

When you're done with the lab, clean up resources to avoid ongoing costs.

1. Stop the Openflow Connector

  1. Navigate to Data → Openflow in Snowsight
  2. Select your PostgreSQL connector
  3. Click Stop to halt the CDC pipeline

2. Clean Up Snowflake Resources

SQL (Snowflake) 
-- Clean up Snowflake resources
USE ROLE ACCOUNTADMIN;

-- Drop the CDC destination database
DROP DATABASE IF EXISTS ECOMMERCE_CDC;

-- Drop the warehouse
DROP WAREHOUSE IF EXISTS OPENFLOW_WH;

3. Clean Up PostgreSQL Resources

SQL (PostgreSQL) 
-- Drop the publication
DROP PUBLICATION IF EXISTS ecommerce_cdc_publication;

-- Drop the schema and all tables
DROP SCHEMA IF EXISTS ecommerce CASCADE;

4. Drop the Postgres Instance

SQL (Snowflake) 
-- Drop the Postgres instance
USE ROLE ACCOUNTADMIN;
DROP POSTGRES INSTANCE IF EXISTS openflow_lab;

What You Accomplished

  • Deployed a fully-managed Snowflake Postgres instance
  • Created an e-commerce schema with relational tables
  • Configured PostgreSQL logical replication for CDC
  • Deployed and configured the Openflow PostgreSQL CDC connector
  • Verified real-time data replication (INSERT, UPDATE, DELETE)
  • Ran analytical queries on the replicated data in Snowflake

Key Takeaways

  • Snowflake Postgres provides a fully-managed PostgreSQL service ideal for transactional workloads
  • Openflow CDC enables real-time data replication with minimal configuration
  • Together, they enable: OLTP in Postgres + OLAP in Snowflake
  • CDC captures all DML operations (INSERT, UPDATE, DELETE) with near-zero latency
