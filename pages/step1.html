Deploy a Snowflake Postgres Instance

Create a fully-managed PostgreSQL instance that will serve as our transactional data source.

Option A: Deploy via Snowsight UI

  1. Log in to Snowsight
  2. Click the + button at the top of the navigation menu, or go to Manage → Postgres
  3. Click Create Instance
  4. Configure with these settings:
SettingValue
Instance Nameopenflow_lab
Instance ClassBurstable — 2 cores, 2 GB memory
Storage10 GB
Postgres Version17 (latest available)
Network PolicyCreate new or use existing (allow your IP)

Option B: Deploy via SQL

SQL (Snowflake) 
-- Use ACCOUNTADMIN role
USE ROLE ACCOUNTADMIN;

-- Create a network rule to allow your IP address
CREATE OR REPLACE NETWORK RULE openflow_lab_network_rule
  MODE = INGRESS
  TYPE = IPV4
  VALUE_LIST = ('0.0.0.0/0');  -- Replace with your specific IP for production

-- Create network policy
CREATE OR REPLACE NETWORK POLICY openflow_lab_network_policy
  ALLOWED_NETWORK_RULE_LIST = ('openflow_lab_network_rule');

-- Create the Postgres instance
CREATE POSTGRES INSTANCE openflow_lab
  INSTANCE_CLASS = 'BURSTABLE'
  CORES = 2
  MEMORY_GB = 2
  STORAGE_GB = 10
  POSTGRES_VERSION = '17'
  NETWORK_POLICY = 'openflow_lab_network_policy'
  AUTO_RESUME = TRUE;
ⓘ
Note

Instance creation typically takes 2-5 minutes. Status changes from PROVISIONING to RUNNING.

Retrieve Connection Credentials

After creation, retrieve your connection details:

SQL (Snowflake) 
-- Check instance status and get connection details
SHOW POSTGRES INSTANCES;
✅ Expected Result

You should see your instance with status RUNNING and a hostname like:
openflow-lab.your-account.us-west-2.aws.postgres.snowflake.app

⚠
Save Your Credentials

Store the connection string and password securely. The default admin user is snowflake_admin. You will need these for both connecting to Postgres and configuring the Openflow connector later.
